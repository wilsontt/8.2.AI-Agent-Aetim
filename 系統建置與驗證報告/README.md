## 驗收報告格式

每個驗收報告包含以下章節：

1. **任務概述**：任務的基本資訊
2. **執行內容**：詳細的實作內容
3. **驗收條件檢查**：逐項檢查驗收條件
4. **測試結果**：測試執行結果
5. **交付成果**：完成的工作項目
6. **相關文件**：相關的設計與任務文件
7. **備註**：其他說明事項

## 報告命名規則

驗收報告檔案命名格式：`{任務編號}-{任務名稱}-驗收報告.md`

範例：
- `T-1-1-1-建立專案結構-驗收報告.md`
- `T-1-2-1-實作資料庫Schema-驗收報告.md`

## 使用說明

1. 每個任務完成後，應立即建立對應的驗收報告
2. 驗收報告應包含完整的驗收條件檢查結果
3. 所有測試結果應記錄在驗收報告中
4. 驗收報告應可追溯至相關的設計與任務文件
5. **建立驗收報告後，必須同步更新本文件的「已完成任務」章節**
